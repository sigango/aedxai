# AED-XAI: Adaptive Explanation-Driven Object Detection

AED-XAI is a research pipeline for **closed-loop, auto-explainable object
detection**. It detects objects, asks a vision-language model to judge detection
quality and scene context, adaptively selects the best explanation method per
detection, evaluates explanation quality without human explanation annotations,
and feeds poor explanation scores back into detector post-processing thresholds.

The central research idea is that explanations should not be generated by a
single fixed XAI method. Instead, AED-XAI learns or heuristically predicts which
method is most appropriate for each detection based on pre-explanation context:
object class, confidence, relative size, scene complexity, crowding, box
geometry, and local image entropy.

## Pipeline

```text
RGB image
   |
   v
+------------------+
| DetectorWrapper  |  YOLOX-S or Faster R-CNN
+------------------+
   |
   v
+------------------+
| VLMJudge         |  Qwen2.5-VL quality, false positive, scene complexity
+------------------+
   |
   v
+------------------+
| XAISelector      |  MLP or rule fallback chooses GradCAM/G-CAME/D-CLOSE/LIME
+------------------+
   |
   v
+------------------+
| XAI Explainer    |  saliency map per detection
+------------------+
   |
   v
+------------------+
| AutoEvaluator    |  PG, EBPG, OA, Sparsity, Composite; no human labels
+------------------+
   |
   v
+------------------+
| FeedbackLoop     |  adaptive thresholding and detector threshold refinement
+------------------+
```

## What Is Implemented

- `DetectorWrapper` for YOLOX-S and torchvision Faster R-CNN.
- `VLMJudge` using Qwen2.5-VL with robust JSON parsing and retry logic.
- XAI methods: GradCAM, G-CAME, D-CLOSE, and LIME for detection.
- `XAISelector` MLP plus rule-based fallback.
- `AutoEvaluator` with annotation-free metrics.
- `AdaptiveThreshold` with fixed, percentile, and learned modes.
- `FeedbackLoop` for threshold refinement.
- `AEDXAIPipeline` for end-to-end orchestration.
- CLI scripts for selector training, AED-XAI experiments, and fixed-method baselines.
- Research notebooks for setup, detection exploration, VLM analysis, XAI comparison,
  results visualization, and full interactive pipeline execution.

## Hardware

Recommended:

```text
Python 3.10+
Linux with NVIDIA GPU
CUDA 12.x or newer driver-compatible runtime
24GB VRAM for RTX 3090/4090 class GPUs
```

Known target platforms:

```text
RTX 3090 / RTX 4090, AMD64, 24GB VRAM
NVIDIA DGX Spark / GB10, ARM64, 128GB unified VRAM
```

For DGX Spark ARM64, use an NVIDIA PyTorch/NGC container if possible. Avoid
letting pip replace the container-provided `torch` and `torchvision` wheels.

## Installation

### Local Python Environment

```bash
python3.10 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
python -m pip install --no-build-isolation -e .
python -m pip install -r requirements.txt
python -m pip install --no-build-isolation -r requirements-yolox.txt
```

YOLOX is installed from GitHub and must be installed after PyTorch is available.
Use `--no-build-isolation` so the YOLOX build can import the already-installed
`torch`.

### DGX Spark / ARM64 Container

```bash
docker run --gpus all -it --rm --ipc=host \
  --ulimit memlock=-1 --ulimit stack=67108864 \
  -v "$PWD":/workspace/aedxai \
  -v "$HOME/.cache/huggingface":/root/.cache/huggingface \
  -v "$HOME/.cache/torch":/root/.cache/torch \
  -w /workspace/aedxai \
  nvcr.io/nvidia/pytorch:25.11-py3 \
  bash
```

Inside the container:

```bash
python -m pip install --upgrade pip setuptools wheel
python -m pip install --no-build-isolation -e .
grep -vE '^(torch|torchvision)' requirements.txt > /tmp/requirements-no-torch.txt
python -m pip install -r /tmp/requirements-no-torch.txt
python -m pip install --no-build-isolation -r requirements-yolox.txt
```

If `bitsandbytes` has issues on ARM64/GB10, set:

```yaml
# config/vlm_config.yaml
quantization: "fp16"
```

## Quick Start

Run one image through the full AED-XAI pipeline:

```python
from src.pipeline import AEDXAIPipeline

pipeline = AEDXAIPipeline(
    detector_config_path="config/detector_config.yaml",
    vlm_config_path="config/vlm_config.yaml",
    xai_config_path="config/xai_config.yaml",
    eval_config_path="config/eval_config.yaml",
)

pipeline.setup()
result = pipeline.run_on_image("data/coco/val2017/000000000139.jpg")
print("Composite score:", result.composite_score)
print("Metadata:", result.metadata)
pipeline.shutdown()
```

Detector-only smoke test:

```bash
python - <<'PY'
from src.detector import DetectorWrapper
from src.utils import load_image

detector = DetectorWrapper("yolox-s", config_path="config/detector_config.yaml")
detector.load_model()
image = load_image("data/coco/val2017/000000000139.jpg")
detections = detector.detect(image)
print("detections:", len(detections))
print(detections[:3])
detector.unload_model()
PY
```

### Selector Training

Small smoke run:

```bash
python scripts/train_selector.py \
  --max-images 20 \
  --target-detections 100 \
  --checkpoint-every 25 \
  --oracle-mode
```

Larger oracle-label generation:

```bash
python scripts/train_selector.py \
  --max-images 500 \
  --target-detections 2000 \
  --oracle-mode \
  --resume
```

Selector training generates oracle labels offline by running all candidate XAI
methods and selecting the method with the best composite score. At inference,
the selector uses only pre-explanation context features.

## Metrics

AED-XAI evaluates explanations without human explanation annotations.

Implemented metrics:

```text
PG      : Pointing Game
EBPG    : Energy-Based Pointing Game
OA      : Insertion AUC - Deletion AUC
Sparsity: Gini concentration of saliency
Composite = equal-weight combination of PG, normalized OA, and Sparsity
```

Current composite weights:

```yaml
pg: 0.333333
oa: 0.333333
sparsity: 0.333333
```

## Common Issues

### YOLOX build fails

Install after PyTorch with build isolation disabled:

```bash
python -m pip install --no-build-isolation -r requirements-yolox.txt
```

### VLM INT4 fails on ARM64

Try FP16:

```yaml
quantization: "fp16"
```

### CUDA is unavailable inside Docker

Check host and container GPU visibility:

```bash
nvidia-smi
docker run --rm --gpus=all nvcr.io/nvidia/cuda:13.0.1-devel-ubuntu24.04 nvidia-smi
```

### Notebook kernel imports wrong environment

Select the same Python environment where `torch`, `pycocotools`, YOLOX, and
AED-XAI are installed.
